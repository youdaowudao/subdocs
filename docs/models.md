# 模型与分组

分组决定一个 API Key 走哪条上游、能用哪些模型、按哪个价格档计费。模型名是客户端实际发给接口的模型 ID。

重要工作用支持高能力模型、价格更高的分组；简单任务和批量小任务用低成本分组。分组不对时，Key 填对也会出现 `403`、`model not found`、模型不可用或计费不符合预期。

## 先分清在哪里改

| 要改什么 | 去哪里改 | 影响什么 |
| --- | --- | --- |
| API Key 分组 | UseGoodAI 后台 **API 密钥** 页面，找到当前 Key，修改分组后保存 | 这个 Key 可用的上游、模型和计费档位 |
| 模型名 | Codex、Cherry Studio、Open WebUI 等客户端配置里修改 | 客户端实际请求哪个模型 |
| API Key | 客户端的 `API Key` / `Bearer Token` 字段 | 请求使用哪个 Key 和哪个分组 |

切换分组后，客户端里仍然要填写新分组支持的模型名。分组改了、模型名没改，客户端还会按旧模型请求。

## 模型怎么选

| 场景 | 先用哪个模型 |
| --- | --- |
| 重要任务、复杂代码、长流程 Agent、专业分析 | `gpt-5.5` |
| 普通工作、日常代码、文档处理、稳定对话 | `gpt-5.4` |
| 简单操作、批量小任务、成本和速度更敏感的场景 | `gpt-5.4-mini` |
| 图片生成、图片编辑、视觉创作 | `gpt-image-2` |

实际能不能调用，以当前 Key 所属分组里的模型列表为准。后台没有显示的模型，不要手写到客户端。

## 切换分组

1. 打开 UseGoodAI 后台的 **API 密钥** 页面。
2. 找到当前客户端正在使用的 Key。
3. 修改这个 Key 的分组并保存。
4. 打开该分组可用模型列表，复制要用的模型名。
5. 回到客户端，把模型名改成该分组支持的模型。
6. 发一条测试请求。
7. 到 **使用记录** 页面确认模型、分组和计费是否符合预期。

分组错误就回到 **API 密钥** 页面改分组。模型错误就回到客户端改模型名。计费有疑问就联系客服核对。

## 常见错误

| 现象 | 先查什么 |
| --- | --- |
| `401` | API Key 是否复制完整、是否填到正确客户端 |
| `403` | 当前 Key 所属分组是否允许这个模型 |
| `model not found` | 模型名是否从当前分组复制，客户端里是否手写错 |
| 后台计费不符合预期 | 使用记录里的模型和分组是否正确 |
