# 模型与分组

分组决定一个 API Key 能调用哪些模型。配置客户端时，模型名以 UseGoodAI 后台当前显示为准，不要照抄其它网站、旧教程或别的账号里的模型名。

## 为什么要看分组

| 项目 | 说明 |
| --- | --- |
| API Key | 发起请求时使用的密钥 |
| 分组 | 限定这个 Key 可用的模型范围 |
| 模型名 | 客户端实际发送给接口的模型 ID |

如果 Key 正确但模型不可用，优先检查这个 Key 所属分组是否支持该模型。不同账号、套餐、分组看到的可用模型可能不同。

## 常见模型

| 模型 | 简单说明 |
| --- | --- |
| `gpt-5.5` | 高能力通用模型，适合复杂推理、代码、Agent 工具调用和专业工作流。 |
| `gpt-5.4` | 通用能力模型，适合代码、文本处理、工具调用和日常高质量任务。 |
| `gpt-5.4-mini` | 轻量模型，适合明确任务、批量处理、自动化流程和对响应速度更敏感的场景。 |
| `gpt-image-2` | 图像能力模型，用于图片生成、图片编辑和视觉创作类任务；接口调用见 [生图 API](/external/image-generation)，画布创作见 [无限画布](/clients/infinite-canvas)。 |

::: tip
上面的模型名只用于说明常见能力。实际能不能调用，仍然以你的 UseGoodAI 后台和当前 API Key 所属分组为准。
:::

## 填模型名的原则

| 原则 | 说明 |
| --- | --- |
| 以后台为准 | 后台没有显示的模型，不要手动照抄到客户端 |
| 区分分组 | 不同 Key 所属分组可能支持不同模型 |
| 先少后多 | 第一次接入先填 1 个确定可用的模型 |
| 原样复制 | 大小写、点号、横线、斜杠都要一致 |

## 正确做法

1. 在后台创建 API Key。
2. 确认这个 Key 所属分组。
3. 在该分组里查看可用模型。
4. 把模型名原样填写到客户端。
5. 先用一个确定可用的模型跑通，再根据速度、效果和成本调整。

## 常见错误

| 错误 | 常见原因 |
| --- | --- |
| 401 | Key 无效、填错或没被客户端读取 |
| 403 | Key 有效，但分组、模型或权限不允许 |
| model not found | 模型名写错，或分组里没有该模型 |
